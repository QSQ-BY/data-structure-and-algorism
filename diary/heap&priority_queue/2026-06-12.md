# 2026-06-12：用 set 和 pair 模拟堆，基础部分收尾

今天把堆这一章的基础部分正式收住了。前几天一直在手写堆：上滤、下滤、建堆、堆排序，再到用小顶堆优化哈夫曼编码。今天换了一个角度，用 `set + pair` 把堆的几个基本操作又过了一遍。感觉这一步挺实用，因为以后写题时，很多地方并不一定要从零实现一个堆，只要能借助现成结构维护最值，就可以直接把精力放到题目本身。

`set` 本身会自动按顺序维护元素，所以它天然适合模拟“小顶堆”的一部分行为。插入元素相当于 `push`，`begin()` 指向当前最小值，相当于堆顶；需要弹出堆顶时，直接 `erase(begin())`。如果要模拟大顶堆，也可以把数值取反后放进去，再在取出时还原。这个办法没有手写堆那么贴近数组里的完全二叉树，但在做题时很省事。

今天比较重要的一点是重复元素的处理。普通 `set<int>` 会去重，两个相同的数只能留下一个，这和堆的行为不一样。堆里如果插入两个 `4`，它们应该都存在。于是我用了 `pair<int, int>`：第一个位置放真正参与比较的值，第二个位置放一个不断递增的编号。这样两个相同的值也会因为编号不同而被保留下来，`set` 又仍然能按照“值优先，编号其次”的顺序排列。

这个写法让我对 STL 里的有序集合和堆之间的关系更清楚了一点。手写堆强调的是父子节点之间的局部关系，`set` 强调的是整棵平衡树维护出来的有序性。它们底层不一样，但在很多题目里都能提供“快速拿到当前最小值或最大值”的能力。以后遇到动态维护最值、允许重复元素、还可能需要删除指定元素的题目时，`set + pair` 就可以成为一个很顺手的工具。

到今天为止，堆的基础内容算是告一段落了。前面先自己实现堆，是为了知道上滤和下滤到底在修什么；今天再用 `set + pair`，就更像是把这部分能力整理成做题时可以直接拿出来用的工具。明天开始进入堆章节的练习，我打算从具体题目里继续看这些性质怎么用。按照现在的节奏，如果下周能把堆题练完，就可以进入下一章排序算法了。
