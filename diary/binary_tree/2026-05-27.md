# 2026-05-27：正式进入树，递归感一下子压上来了

今天正式进入树这一章了。

昨天表达式计算最后已经摸到了一点树的影子：一个表达式可以拆成根运算符、左子表达式和右子表达式。但今天开始写二叉树的基础定义和结构操作时，那种感觉变得更直接了。之前在顺序表、链表、栈和队列里，数据结构大多还是一条线，哪怕链表有指针，脑子里也能顺着 next 一路往后走。树不一样。一个节点下面分出左孩子和右孩子，函数一递归下去，执行过程就开始在脑子里分叉。

今天主要写的是二叉树最基本的一套东西：节点结构体里有 `data`、`lchild` 和 `rchild`，`get_new_node` 负责申请并初始化节点，`insert` 用递归把新节点插到树里的某个空位置，后面又补了层序遍历的 `bfs`、深度优先遍历的 `dfs`，最后用 `clear` 递归释放整棵树。内容看起来都很基础，但写起来明显比前面的线性结构更容易让人短暂停住。

最让我反复想的是 `insert`。一开始它那句

```cpp
root->lchild = insert(root->lchild, data);
```

看起来有点绕：明明是往左子树里插入，为什么还要重新赋值给 `root->lchild`？后来慢慢想清楚了，递归函数返回的不是随便一个节点，而是“插入完成后的这棵子树的根”。如果左孩子本来是空，递归会创建一个新节点并返回，于是这句赋值就把新节点接到了父节点下面；如果左子树本来不空，递归最后还是返回原来的左子树根，重新赋值也不会破坏原结构。

这个理解挺关键。它让我第一次比较清楚地感到，递归里的 `root` 不是永远指整棵树的根，而是每一层调用当前负责处理的那棵子树的根。最深处遇到 `nullptr` 时创建新节点，然后一层层返回：新节点返回给它的父节点，父节点返回给上一层，最后最外层返回整棵树的根。这个过程如果只看代码，很容易觉得它像是“凭空回来了”；但把每一层当成一棵子树来看，就顺了很多。

`bfs` 这部分反而更像前面队列知识的复用。用数组模拟队列，`head` 出队，`tail` 入队，先把根节点放进去，再不断取出当前节点，把它的左右孩子加入队列。它的思路和之前学过的队列操作是一脉相承的，只是队列里装的不再是普通数字，而是一个个节点指针。这里能感觉到前面的基础没有白学，树的遍历只是把旧工具换了一个更复杂的使用场景。

`dfs` 又把递归的味道拉满了。今天这个版本用全局变量 `tot` 记录进入和离开节点的时间：第一次走到某个节点时记 `start`，左右子树都递归完以后再记 `end`。最后输出的是每个节点的区间 `[start, end]`。这个设计很有意思，因为父节点的区间会包住子节点的区间，像是在树上给每个递归过程画了一个边界。代码不长，但要真正明白它为什么是先进入、再递归、最后离开，还是得在脑子里跟着函数栈走一遍。

今天只完成了二叉树的基础结构定义与结构操作，却已经明显感觉难度上来了。树这一章不是单独冒出来的新东西，它把前面很多内容都拧到一起：链表的指针，队列的层序遍历，递归的调用与返回，还有一点点对函数栈的直觉。以前我可能会把这些知识点分开放，现在发现它们在树里会一起出现，而且缺一个都不太稳。

也因为这样，今天有种“道阻且长”的感觉。刚进入树，还只是最基础的二叉树，就已经需要不断回头确认递归到底在干什么。后面如果继续往二叉搜索树、线索化、堆、哈夫曼树之类的方向走，应该会更吃对结构的理解，而不是只会照着模板写几行代码。

偏偏马上又要进入期末阶段了。数据结构这边的学习时间大概率会被压缩，刚开起来的俄罗斯方块新坑也可能要先搁置一阵。多少有点不舍，毕竟那个项目刚有点新鲜劲。但现在更现实的问题是：树这一章不能只混个眼熟就过去。今天的代码虽然只是起步，至少让我看清楚了后面要补的东西：递归调用的返回值、子树视角、遍历顺序，还有如何把前面学过的结构真正接到新的问题里。

今天就先走到这里。树这扇门算是推开了，但里面显然不是一条直路。接下来要在期末复习和数据结构之间挤时间，能挤多少算多少，关键是别把今天刚建立起来的递归感觉放凉。
