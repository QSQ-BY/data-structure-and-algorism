# 2026-06-03：四道二叉树简单题，跟着写也不是白写

今天写了四道 LeetCode 上的树题：二叉树层序遍历、从前序和中序遍历还原二叉树、翻转二叉树，还有 N 叉树的前序遍历。

说实话，因为时间比较紧，这几道题基本都是跟着老师写的。题目本身也属于比较应试的简单题，不是那种会逼着我从头设计很久的题。但做完以后也不能说毫无收获。至少我又把二叉树这一章最基础、最容易在题里反复出现的东西过了一遍。

## 遍历这件事又清楚了一点

今天最明显的感觉是，前序、中序、后序遍历和层序遍历终于能放在同一张图里看了。

前序、中序、后序本质上都在往深处走，所以属于 DFS。区别只是访问根节点的时机不同：前序先处理根节点，中序把根节点放在左右子树中间，后序则等左右子树都处理完以后再处理根节点。

层序遍历换了一个方向。它不急着沿着某一条分支走到底，而是一层一层访问，所以用队列来写最自然。写 `lc102_levelorder.cpp` 的时候，我同时看到了 BFS 的队列写法和 DFS 带层数参数的写法。两种都能得到层序结果，但脑子里的模型不一样：队列是在模拟一层一层推进，DFS 则是递归时顺手把当前层数带下去。

这部分以前我也听过，但今天配着代码再看，印象更实在一点。

## 二层封装不只是为了好看

今天另一个比较有用的点，是 N 叉树前序遍历里的二层封装。

一开始我会觉得，既然 LeetCode 要求的是：

```cpp
vector<int> preorder(Node* root)
```

那能不能直接在这一个函数里递归完成？当然可以，但如果每次递归都返回一个新的 `vector<int>`，再把子树结果复制回父节点，效率会变差。每棵子树都生成临时数组，父节点还要重新搬一遍，树越大，重复拷贝越明显。

老师写的第一种方法用了一个辅助函数：

```cpp
void preorderDfs(Node* root, vector<int>& ans)
```

这个写法把 `ans` 作为引用参数传下去。整棵树遍历过程中，结果数组只创建一次，每个节点访问到就直接写进去。今天我才更清楚地意识到，二层封装不只是为了把代码拆开、让主函数看起来整洁，它还可以让递归函数多带一个“持续存在的状态”，从而少掉一堆没有必要的拷贝。

这和前几天写二叉树转广义表时的感觉有点接上了。外层函数负责准备环境，真正递归的函数负责处理当前节点和子树。只不过今天这个例子里，二层封装的收益更直接地体现在效率上。

## 简单题也能查出基础稳不稳

四道题里，翻转二叉树最直观：交换左右孩子，然后递归处理左右子树。层序遍历考的是队列和每层节点数量。从前序和中序还原二叉树，则要求我真的理解前序的第一个元素为什么是根，以及中序数组为什么能把左右子树切开。

这些题都不难，但它们会反复问同一件事：我能不能把“当前节点”和“当前子树”分清楚。递归写起来短，前提是心里知道这个函数到底在处理哪棵树。如果只是照着模板敲，代码能跑，但换个题就容易没底。
