## 顺序表：先把连续空间管明白

顺序表是我这段学习里最早搭起来的基础结构。它看起来就是一个数组外面包了一层结构体：

```cpp
typedef struct Vector
{
    int *data;
    int size;
    int count;
} vector;
```

但真正写起来，重点不在“数组能存数字”，而在于 `size` 和 `count` 的区别。

`size` 是容量，说明这块连续空间最多能放多少元素。`count` 是当前已经用了多少。很多边界判断都绕不开这两个值，比如插入前要判断空间够不够，删除时要判断位置是不是落在已有元素范围内。

我给顺序表写了这些基础操作：

- `create_vector`：申请结构体和底层数组。
- `expand`：空间不够时用 `realloc` 扩容。
- `insert`：在指定位置插入元素，后面的元素整体右移。
- `erase_position`：按位置删除，后面的元素整体左移。
- `erase_number`：按值删除，遇到多个相同元素时要注意下标回退。
- `change_position` 和 `change_number`：按位置或按值修改。
- `destroy_vector`：释放数组和结构体。

最容易写错的是删除。按位置删除比较直接，从 `pos` 开始把后一个元素往前覆盖就行。按值删除麻烦一点，因为删除一个元素后，后面的元素会左移。如果循环下标继续往前走，就可能漏掉连续出现的目标值。后来我在删除后写了 `i--`，这个小地方比它看上去重要。

顺序表练习里，我还写了几类典型题。

二分查找让我重新确认了区间定义的重要性。代码里采用的是左闭右闭区间 `[left, right]`，所以循环条件是 `left <= right`，移动边界时也要写成 `right = mid - 1` 和 `left = mid + 1`。如果区间定义没想清楚，二分很容易变成看起来对、实际卡边界的代码。

快慢指针用在原地删除元素和有序数组去重上。慢指针记录下一个可写位置，快指针负责往前扫描。这个思路挺干净：不是每次真的删除，而是把要保留的元素覆盖到前面。

滑动窗口用在长度最小的连续子数组题里。右指针负责扩大窗口，窗口和满足条件后，左指针再往右收缩。这个题让我第一次比较明确地感受到：双指针不是两个变量随便跑，而是一个负责扩张，一个负责修正。

前缀和则是另一种思路。它提前把区间信息存起来，后面求 `[a, b]` 的区间和时，只要用 `prefix_sum[b + 1] - prefix_sum[a]`。这不像双指针那样动态移动，更像是用一次预处理换后面多次查询的速度。

平方排序那题也挺有意思。数组本来有序，但平方后最大值可能出现在两端。于是我从结果数组的尾部开始填，每次比较左右两端的平方，把大的放到后面。这里又回到了双指针，不过这次不是删除，也不是窗口，而是在有序条件下从两端取值。

## 链表：真正要管的是指针关系

链表比顺序表绕一点。顺序表的问题大多在下标，链表的问题大多在指针。

我先写了最基础的节点：

```cpp
typedef struct node{
    int val;
    struct node* next;
}node;
```

然后是创建节点、插入、删除、查找、打印和销毁。插入时我写了两个版本，一个是头节点单独讨论，一个是使用虚拟头节点。

虚拟头节点这个写法对我挺有帮助。以前处理头插、删头节点时，总要单独写一段逻辑。加一个 dummy head 后，很多操作都变成“处理前一个节点的 next”。代码不一定短很多，但脑子里少了一种特殊情况。

链表基础里还顺手做了循环链表和双向链表。

循环链表的关键是尾节点的 `next` 指回头节点。打印时不能再用普通链表那种 `while(p)`，不然就停不下来，所以我用计数限制输出次数。双向链表多了 `pre` 指针，正向遍历和反向遍历都能走，这让我更直观地看到：多一个指针字段，结构能力变强了，维护成本也跟着变高。

链表练习题里，反转链表是最核心的一类。

我写了三种反转方式：头插法、三指针法和递归法。头插法像是不断把旧链表里的节点摘下来，插到新链表头部。三指针法更直接，`pre`、`cur`、`temp` 三个指针轮流推进，把每个节点的 `next` 反过来。递归法最不直观，但想明白后很漂亮：先把后面的链表反转好，再让后一个节点指回当前节点。

删除倒数第 n 个节点这题让我又碰到了虚拟头节点。直接算长度也能做，但 dummy head 加快慢指针的写法更稳定，尤其是删除头节点时，不需要额外分支。

判断链表是否有环，用的是快慢指针。快指针一次走两步，慢指针一次走一步。如果有环，它们一定会相遇。找环入口时，相遇后让一个指针回到头节点，两个指针再一起走，它们第二次相遇的位置就是入口。这个结论第一次看有点像魔法，写出来以后反而更容易接受。

旋转链表则让我意识到，很多链表题其实是在找“断点”。先算长度，把 `k` 对长度取模，再找到新的尾节点，把链表断开并接回去。链表操作不是移动值，而是改连接关系。

还有一个快乐数题，表面上不是链表，但我用了同样的“环”思路。一个数不断变成各位数字平方和，如果最后不是 1，就会进入循环。于是它也可以看成一个隐形链表：每个数字指向下一个数字。快慢指针照样能判断循环入口。

这一点挺有意思。算法题里的“链表”有时候不一定真的长得像链表，只要状态能不断跳到下一个状态，就可能有链表里的环。

## 递归：把当前这一步写清楚

递归函数这一块，我从阶乘开始。

阶乘的代码很短：

```cpp
int factorial(int n)
{
    if (n == 0 || n == 1)
    {
        return 1;
    }
    return n * factorial(n - 1);
}
```

短归短，但它把递归最基本的两个东西摆出来了：出口和递推关系。没有出口，递归停不下来。没有递推关系，递归也不知道怎么把大问题变小。

后面我写了排列、组合和子集枚举，这几题比阶乘更能练递归的手感。

排列枚举里，我用 `flag` 数组记录某个数字是否已经用过。每一层递归都在当前位置选一个还没用过的数字，进入下一层，回来后再把标记恢复。这个“选、递归、撤销选择”的过程，就是回溯的雏形。

组合枚举不一样。组合不关心顺序，所以每一层不需要从头开始选，而是从上一次选择数字的后面继续。代码里用 `min` 控制下一层能选的最小值，这样就不会出现 `1 2 3` 和 `2 1 3` 这种重复结果。

子集枚举更像是在不断选择“接下来放哪个数字”。每选到一个数字就可以先打印当前结果，再继续往后扩展。它不像排列那样要求填满固定长度，也不像组合那样只打印固定长度的结果。

这三类枚举放在一起看，差别就清楚了：

- 排列：位置重要，不能重复使用数字。
- 组合：顺序不重要，只往后选。
- 子集：每个中间状态本身就是一个结果。

递归写多了以后，我慢慢不再把它看成“函数自己调用自己”这么抽象的东西。更实用的想法是：当前这一层负责做一个选择，把剩下的交给下一层；下一层回来后，当前层再恢复现场，继续尝试别的选择。

## 这段时间的收获

这段学习把几个东西分开了。

顺序表关心连续空间。插入、删除和扩容都围绕数组下标和元素移动展开。

链表关心指针关系。它不需要移动一大片元素，但每一次修改连接都要小心，尤其是头节点、尾节点、空链表这些边界。

递归关心问题怎么拆。只要能找到出口，能说明当前层做什么，剩下的部分就可以交给下一层。

现在再看后面的结构，感觉心里有了底。不是因为每个函数都写得完美，而是我开始知道自己在处理哪一种关系：连续空间、节点连接，还是一层一层展开的问题。

这篇补记也算给前面几天留个锚点。之后再翻到这些文件，我至少能想起来，当时不是单纯在刷代码，而是在一点点把数据结构的基本语感搭起来。
