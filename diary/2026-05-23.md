# 2026-05-23：两道练习，从栈的熟练到队列题的抽象

今天的代码量不算多，只做了栈和队列的两道练习，下午打算多学学高数。但写完以后感觉收获并不少，尤其是第二题，刚看到题面的时候我确实没什么思路，听完老师把逻辑拆开之后才发现，它真正难的地方不在代码，而在把题目翻译成一个能处理的模型。

## 函数调用链：括号匹配的另一个版本

第一题是函数调用关系。题目给出一串简化后的代码行，每一行要么是一次函数调用，比如 `fun()`，要么是 `return`。要求找到某个目标函数第一次出现时的调用链。

这题我写得比较顺，因为它和前面做过的括号匹配几乎是同一个结构。

遇到函数调用，就把函数名压栈；遇到 `return`，说明当前函数结束了，就从栈里弹出。目标函数第一次出现时，栈里保存的正好就是从最外层到当前函数的调用链。

代码里核心逻辑其实很短：

```cpp
if (ops[i] == "return") {
    if (!stack.empty()) {
        stack.pop_back();
    }
} else {
    stack.push_back(ops[i]);
    if (ops[i] == target) {
        // 输出当前调用链
    }
}
```

这让我又把“栈适合处理嵌套关系”这件事过了一遍。括号匹配里，左括号打开一层，右括号关闭一层；函数调用里，调用打开一层，`return` 关闭一层。换一个题面，结构没有变。

所以这题对我来说更像是一次熟练度练习。看到题目的时候能比较快地想到栈，写起来也没有太多阻力。前几天学的东西没有停在“我知道栈是什么”，而是开始能在新题里自己冒出来了。

## 三个集合的最小距离

第二题就有意思多了。

题目给三个升序集合，从里面各取一个数 `a`、`b`、`c`，要求让

```text
D = |a-b| + |b-c| + |c-a|
```

尽可能小。

一开始看这个式子，我会自然地想枚举，或者想怎么在三个集合之间找组合。但如果真的三重循环，复杂度肯定不理想。题目又说三个集合已经升序存储，这个条件不可能是白给的。

老师讲完之后，最关键的一步是把这个距离放回数轴上看。三个数不管顺序怎样，设其中最小值是 `min`，最大值是 `max`，中间那个数夹在中间。那这个距离其实就是：

```text
D = 2 * (max - min)
```

也就是说，题目表面上在算三项绝对值，实际上是在问：怎么让当前三个数的最大值和最小值尽量接近。

这个抽象一出来，队列操作就顺了。每个集合都是升序的，队头就是当前能选的候选值。每一轮拿三个队头算一次距离，然后移动最小值所在的那个队列。

为什么移动最小值？因为当前距离由 `max - min` 决定。最大值继续往后走只会更大，中间值往后走也不一定能缩小范围。真正有机会缩小区间的，是让当前最小值往后挪，去追另外两个数。

所以策略可以压成一句话：

```text
每次比较三个队头，记录答案，然后弹出最小值所在队列的队头。
```

这题虽然放在队列练习里，但我感觉最难的不是 `front()` 和 `pop()`，而是把数学式子、数轴关系、有序集合这几个信息接起来。代码只是最后那一步。

## 今天的感觉

今天两道题放在一起，还挺能看出差别。

第一题是我已经比较熟的模型：嵌套关系，栈，进一层出一层。只要题意看懂，代码自然就出来了。

第二题则是先卡在理解上。它看起来是队列题，但真正的入口是距离公式。只有先意识到 `D` 等价于两倍的最大差值，后面才知道为什么要盯着三个队头，为什么每次移动最小值。

这大概也是算法题最磨人的地方：有时候数据结构只是工具，题目真正考的是能不能把一段绕着说的话抽成一个清楚的不变量。今天这道考研题我自己一开始想不出来，这没什么好装的。但听懂之后会有一种“哦，原来卡点在这里”的感觉。

下一次再遇到这种题，我应该先少急着写代码，多问一句：这个式子或者条件能不能换一种说法？如果能把题目从原来的语言里拎出来，后面的队列、栈、指针才有地方落脚。
